# 📖 BaziAI API 使用说明

## 🎯 项目概述

这是一个 BaziAI 的 API 接口项目，提供 Web 界面和 API 端点来调用 BaziAI 服务。

## 🚀 快速开始

### 方案 1：本地使用（推荐，最简单）

1. **启动服务**：
   ```bash
   python start_local.py
   ```
   或者：
   ```bash
   python app.py
   ```

2. **访问地址**：
   - 主页：http://127.0.0.1:5000
   - 测试页面：http://127.0.0.1:5000/test

3. **输入 Cookie**：
   - 点击"加载默认 Cookie"按钮
   - 或手动粘贴你的 Cookie

4. **开始使用**！

### 方案 2：公网访问（需要 ngrok）

如果你想从外网访问，需要设置 ngrok：

1. **查看设置指南**：
   ```bash
   阅读 NGROK_SETUP.md 文件
   ```

2. **注册并配置 ngrok**（一次性设置）

3. **启动服务**：
   ```bash
   python start_with_ngrok.py
   ```

4. **使用公网 URL 访问**

## 📁 重要文件

| 文件 | 说明 |
|------|------|
| `app.py` | Flask 主应用 |
| `start_local.py` | 本地启动脚本（推荐） |
| `start_with_ngrok.py` | ngrok 启动脚本（公网访问） |
| `bazi_credentials.json` | 保存的 Cookie 信息 |
| `templates/index.html` | 主页面 |
| `templates/test.html` | 测试页面 |

## 🌐 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 主页 |
| `/test` | GET | 测试页面 |
| `/health` | GET | 健康检查 |
| `/api/login` | POST | 登录（保存 Cookie） |
| `/api/send` | POST | 发送消息 |
| `/api/messages` | GET | 获取消息列表 |
| `/api/create-session` | POST | 创建新对话 |
| `/api/create-and-send` | POST | 创建对话并发送消息 |

## 💡 使用示例

### 1. 登录

```bash
curl -X POST http://127.0.0.1:5000/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "26a7d080-283c-43c9-a741-23d8dfcb8512",
    "cookie": "你的完整Cookie"
  }'
```

### 2. 发送消息

```bash
curl -X POST http://127.0.0.1:5000/api/send \
  -H "Content-Type: application/json" \
  -d '{"content": "你好"}'
```

### 3. 获取消息

```bash
curl http://127.0.0.1:5000/api/messages
```

## ⚠️ 重要说明

1. **AI 回复查看**：
   - AI 的回复只能在 BaziAI 官网查看
   - 访问：https://www.bazi-ai.com/zh/chat/你的session_id
   - API 只能发送消息，不能获取 AI 回复

2. **Cookie 有效期**：
   - Cookie 会过期，需要定期更新
   - 运行 `python get_cookie_v2.py` 获取新 Cookie

3. **云端部署问题**：
   - Railway 服务器无法访问 BaziAI（地理限制）
   - 建议使用本地服务器

## 🔧 故障排除

### 问题 1：无法启动服务

```bash
# 检查端口是否被占用
netstat -ano | findstr :5000

# 杀死占用端口的进程
taskkill /PID <进程ID> /F
```

### 问题 2：Cookie 失效

```bash
# 重新获取 Cookie
python get_cookie_v2.py
```

### 问题 3：连接失败

- 确保本地服务器正在运行
- 检查防火墙设置
- 确认 Cookie 有效
