## 🚀 下一步：立即部署

### 方案 1: Render（最推荐）⭐⭐⭐⭐⭐

#### 为什么选择 Render？
- ✅ **完全免费**（无需信用卡）
- ✅ **5 分钟部署**
- ✅ **自动 HTTPS**
- ✅ **持续部署**（代码更新自动部署）
- ✅ **支持 Python/Flask**

#### 立即开始

**步骤 1**: 访问 Render
```
https://render.com
```

**步骤 2**: 使用 GitHub 登录
- 点击 "Sign in with GitHub"
- 授权 Render 访问你的仓库

**步骤 3**: 创建 Web Service
1. 点击 **"New +"** 按钮
2. 选择 **"Web Service"**
3. 找到 `bazi` 仓库
4. 点击 **"Connect"**

**步骤 4**: 配置服务

Render 会自动读取 `render.yaml`，但请确认以下配置：

```
Name: bazi-ai-api
Environment: Python 3
Region: Singapore (⚠️ 重要！必须选择亚洲节点)
Branch: main
Build Command: pip install -r requirements.txt
Start Command: gunicorn app:app --bind 0.0.0.0:$PORT
```

**⚠️ 重要提示**：
- **必须选择 Singapore 或 Hong Kong 区域**
- 美国/欧洲节点无法访问 BaziAI API

**步骤 5**: 点击部署

点击 **"Create Web Service"**

等待 5-10 分钟，Render 会：
1. 📦 克隆你的 GitHub 仓库
2. 🔨 安装 Python 依赖
3. 🚀 启动应用
4. ✅ 部署完成

**步骤 6**: 获取 URL

部署成功后，你会得到类似这样的 URL：
```
https://bazi-ai-api.onrender.com
```

或者：
```
https://bazi-ai-api-xxxx.onrender.com
```

## 🐛 常见问题

### Q1: 部署失败怎么办？

**A**: 查看 Render Dashboard 的 Logs 标签，找到错误信息。

常见原因：
- requirements.txt 格式错误
- Python 版本不兼容
- 依赖安装失败

解决方法：
```bash
# 本地测试
pip install -r requirements.txt
python app.py
```

### Q2: 无法访问 BaziAI API？

**A**: 检查区域设置。

**重要**：必须选择 **Singapore** 或 **Hong Kong** 区域！

美国/欧洲节点无法访问 BaziAI。

### Q3: 应用响应很慢？

**A**: 这是免费套餐的休眠机制。

解决方案：
1. 配置 UptimeRobot（推荐）
2. 升级到付费套餐（$7/月，无休眠）

### Q4: 如何查看日志？

**A**: 在 Render Dashboard：
1. 进入你的服务
2. 点击 **"Logs"** 标签
3. 查看实时日志
